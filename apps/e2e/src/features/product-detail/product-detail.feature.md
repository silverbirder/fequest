# フィーチャ: プロダクト詳細ページを E2E で検証する

## シナリオ: 管理画面で登録したプロダクト詳細をユーザーが閲覧できる

![product detail](./product-detail-screenshot.png)

- 前提 admin アプリケーションのコンテナを起動している
- かつ user アプリケーションのコンテナを起動している
- かつ 管理画面からサンプルのプロダクト名・ロゴ・公式サイト・説明文を保存している
- もし プロダクトページを開いたとき
- ならば 登録したプロダクト名・ロゴ・公式サイト・説明文を確認できる
- かつ Playwright でプロダクトページにアクセスしてスクリーンショットを保存できる

## シナリオ: 認証したユーザーがフィーチャーリクエストを投稿する

![product detail auth](./product-detail-auth-screenshot.png)

- 前提 admin アプリケーションのコンテナを起動している
- かつ user アプリケーションのコンテナを起動している
- かつ 管理画面からサンプルのプロダクトが登録されている
- かつ 認証済みのユーザーセッションが存在する
- もし 認証ユーザーとしてプロダクトページで "E2E 新規フィーチャー" を投稿する
- ならば 投稿したフィーチャーリクエストが一覧に表示される
